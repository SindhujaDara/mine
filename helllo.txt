What is Git?
Git is a distributed version control system used to track changes in source code.
It lets you:
Save versions of your project
Go back to previous versions
Work on branches
Collaborate without overwriting each other's work
Git works on your local computer.

GitHub?
GitHub is an online platform that hosts Git repositories.
It lets you:
Store your Git repositories in the cloud
Collaborate with others
Review code
Manage projects (issues, pull requests, etc.)
GitHub is a website/service that works using Git.

Git	GitHub differences
Type	Version control tool	Hosting & collaboration platform
Where it works	Local computer	Online (cloud)
Purpose	Track project history	Store/share Git repositories
Works without the Internet?	Yes	No
Created by	Linus Torvalds	Microsoft (owns GitHub)

Git Commands (Used in Terminal)
Basic Commands
Command	Description
git init	Create a new Git repository
git clone <url>	Download a repository
git status	Check current repo status
git add <file>	Stage a file for commit
git add .	Stage all files
git commit -m "message"	Save changes
git log	Show commit history

Branching

| Command                 | Description                        |
| ----------------------- | ---------------------------------- |
| `git branch`            | List branches                      |
| `git branch <name>`     | Create a new branch                |
| `git checkout <branch>` | Switch branches                    |
| `git merge <branch>`    | Merge a branch into current branch |

Remote github

| Command                       | Description                       |
| ----------------------------- | --------------------------------- |
| `git remote add origin <url>` | Connect to GitHub repo            |
| `git push -u origin main`     | Push code to GitHub               |
| `git pull`                    | Download latest changes           |
| `git fetch`                   | Check for updates without merging |

What is Source Code Management (SCM) on GitHub?
Source Code Management (SCM) on GitHub means using GitHub to:
Store your source code
Track all changes
Collaborate with others
Review and merge updates
Manage versions of your project
GitHub acts as an online storage + collaboration system for your Git repositories.

How Source Code Management Works on GitHub
1. Create a repository (repo)
A repo is like a folder that stores your code and history.
2. Push your code to GitHub
Upload your project using Git.
3. track changes
Every update you make is stored as a commit.
4. Work with branches
Branches allow separate development without messing up the main code.
5. Create Pull Requests (PR)
Ask to merge branch changes into the main project.
6. Merge changes
Approved PRs are merged into the main branch.
7. Collaborate with team
Teams can comment, review, and contribute.

.Fetch & Pull

* CI/CD (Continuous Integration & Continuous Delivery)
* Automating builds, tests, and deployments

2. Install Jenkins on Ubuntu

Step 1: Update system
sudo apt update
sudo apt upgrade -y

Step 2: Install Java
sudo apt install openjdk-17-jdk -y

 Step 3: Add Jenkins repository

curl -fsSL https://pkg.jenkins.io/debian-stable/jenkins.io-2023.key | sudo tee \
  /usr/share/keyrings/jenkins-keyring.asc > /dev/null
echo deb [signed-by=/usr/share/keyrings/jenkins-keyring.asc] \
  https://pkg.jenkins.io/debian-stable binary/ | sudo tee \
  /etc/apt/sources.list.d/jenkins.list > /dev/null

Step 4: Install Jenkins
sudo apt update
sudo apt install jenkins -y

Step 5: Start & enable Jenkins
sudo systemctl start jenkins
sudo systemctl enable jenkins

Step 6: Open Jenkins in browser
http://your-server-ip:8080
http://localhost:8080

Step 7: Get initial admin password
sudo cat /var/lib/jenkins/secrets/initialAdminPassword

Step 8: Install plugins & create admin user
Done!
